Portfolio - Case Study

All the advice suggests you should be honest, to let the recruiter know your thought process. And that is the core of this case study.

Note

When started this project, I always write my ideas on a drawing sketchbook.
My dumb ass didn't think of capture it or store it digitally, and one day, the sketchbook drop on a bucket of water, So this case study missing a lot of images about the process.
I'll do my best to describe it in the text and related pictures.

The Goal

- Create a unique Portfolio to demonstrate my skills.

- Practice, Practice, Practice.

- Embrace failure.

Research

I think this is the most fun part. I spent days browsing Awwwards.com for inspiration and ideas.
The most memorable sites usually have these tag:
Web and Interactive, Clean, Minimal, 3D, Animation.

After that, I discover the Awwwards winner of the year sites. I noticed from 2018 to 2019, 3D trend are rising, winner sites that use WebGL go from 5/7 to 7/7.

Objective

After lots of notes, insight, inspiration, The questions start to come up:
- What make a website memorable?
- What action for the user?
- How to keep the user engaged?

And I find these to be the objective:
An interactive, 3d experience, with minimal and clean design.

First Idea

The first thing I thought of is to use Threejs and particle effect.
My original design is the cursor will react with the surrounding particle, suck in, pull out. The effect, particle, background change along with page position.

The main inspiration for this is squilla.io. Made by RedCollar Agency.
Of course, Mine couldn't be this beautiful. It is much worse.

First Idea - Rejected

After finished the first scene, I noticed there a lot of disadvantages when using particle: isn't suit for mobile, there not much meaning to it, get bored really fast.
And after show it to my friend, the result is indeed correct. The scene only able to keep them engaged for 10 seconds, and there nothing memorable.
It has great potential, but I don't know how to bring it out.

Second Idea - Rejected

My second idea comes and goes really fast, Using CannonJS, a physic library, I simulate a fire extinguisher, you control the pipe using the mouse to shoot smoke and it will push other objects like a box, and sphere around it.
It's fun, I play, test with it for hours. my friend having fun with it too.
But the problem appears when I check the Task Manager, It uses more than 35% CPU on my i5-8400, 30% GPU GTX-1060.
Without a doubt no way this page gonna work smoothly on mobile.

Third Idea - Final

After 2 rejected ideas, I got a lot better at narrowing the scope, how to plan the full-page experience from start to end.
The ideas simple, an amusement park, I make animation for the 3d model, and it plays when the user clicks. and it works.

It's amazing how when you got the foundation right, ideas for the rest of the sites just come naturally, The detail of the user flow map keeps getting clearer.

Challenge & Lesson

The scope of this project is too much, I can't keep up with it, Lots of things are new, I have to learn from scratch. After getting out of my comfort zone and started asking more questions, more people to help. I quickly realized:
The fastest way to learn something is to learn from others.

And all my struggles with 3D model, design, web performance is resolved one after another.

Summary

- 3D model too much polygons, heavy
=> Low poly design, art style becomes more consistent, lighter model, good performance.

- Animation, the physic library can't run smoothly at 60fps
=> TweenMax, requestAnimationFrame, now it runs smoothly and scales with the device refresh rate.

- The default lighting and shadow use too much CPU, GPU
=> Use matcap and shadow textures.

Result

The site complete with all my original objectives. But I don't know how to get feedback. I will keep on learning.

I hoped this Portfolio proved how much passion I have for Web development, The length I would go for a good product.
